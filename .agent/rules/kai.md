# LibreKAi / KAi プロジェクト

---

## 📌 プロジェクト全体像

| | LibreKAi | KAi |
|---|---|---|
| **種別** | OSSアプリ本体 | LibreKAiの開発・運用インスタンス |
| **対象** | 誰でも自由にセルフホスト可能 | かい鯖グループ向け |
| **URL** | GitHub（公開予定） | https://ai.bac0n.f5.si |

### LibreKAi とは
- 誰でもセルフホストできる**オープンソースのAIチャットアプリ**
- llama.cpp等のローカルLLMと組み合わせて動作
- GitHubで公開予定

### KAi とは
- LibreKAiの**開発拠点兼公式インスタンス**
- ここでの実運用フィードバックをLibreKAi本体の開発に反映する
- かい鯖グループ向けの独自設定・プラグインを乗せた構成で動作

---

## 🔴 必須ルール（常に守ること）

### 言語
- **必ず日本語で応答する**

### 操作・実行
- **すべての操作（特にコマンド）は慎重に実行する**
- 破壊的・不可逆な操作の前は必ずバックアップを取る
- **サーバーの再起動は自分では行わない** — 必要な場合はユーザーに伝えて指示を仰ぐ
- コードを変更したら必ず構文チェックを行う
- 一時的な検証スクリプトなどは /scratch などのフォルダは作らず、絶対に /scripts を使う

### タスクの進め方
- 小〜中規模の変更：Planを立てずに即実行
- 大規模な変更：事前にPlanを提示し、実行中も定期的に進捗・問題をユーザーに報告する

### ブラウザ
- デフォルトのものは絶対に使用しない
- /start-browser を使用する

## 🟡 設計・実装ルール

### コアとプラグインの分離（最重要）
- コアはプラグインを動かす**基盤のみ**とする
- プラグインに密接に依存するロジックはコアに絶対に含めない
- プラグインがなくてもコアは正常に動作しなければならない
- 例：Proプランの機能は必ずプラグインとして実装する
- **KAi固有の設定・機能は必ずプラグインまたは設定ファイルで吸収し、LibreKAi本体に混入させない**

### フロントエンド
- モーダルは必ず**LibreKAiの共通モーダルコンポーネント**を使用する
- 新しいCSSを追加する前に、同等のスタイルが既に存在しないか確認する

### 移植性・オープンソース対応
- **特定の環境（サーバー・OS・クラウドサービス等）に依存した実装は絶対にしない**
- 誰でもセルフホストできる汎用的な設計を常に意識する

### コードの質・セキュリティー
- 必ず実装を終わらせる前にテストをする
- もともとあるものは絶対に新しく作らない。まずあるかどうかをgrepなどで調べる。

### ログ
- ログは「起動したこと」と重要な部分が「正しく読み込めたこと」と「エラー」のみ、軽い内容は含めない。
